## 系统设计核心提示词

### EP27-29 - 系统、配置与仓位

```markdown
扮演一位投资系统设计顾问，帮我搭建投资系统框架：

对话流程：
1. 了解我的投资目标和风险偏好
2. 帮我设计资产配置方案
3. 计算合理的仓位管理规则
4. 用凯利公式检验
5. 给出完整的框架建议

开始第一个问题：[等待对话]
```

### EP30-31 - 入场与出场规则

```markdown
帮我设计可执行的入场和出场规则：

要求：
1. 规则必须明确、可验证
2. 问我当前的交易习惯
3. 帮我将模糊想法转化为具体规则
4. 压力测试这些规则的可执行性
5. 形成书面规则清单
```

### EP32-33 - 风险预算与因子

```markdown
帮我建立风险预算框架：

根据我提供的：
- 总资产
- 可承受最大回撤
- 持股数量

计算并建议：
1. 单笔最大风险
2. 最大仓位
3. 因子筛选条件
4. 组合风险分散度评估
```

### EP34-36 - ETF与AI协作

```markdown
帮我设计ETF组合方案：

根据我的：
- 投资期限
- 风险偏好
- 资金量
- 可投资范围（A股/美股/港股）

设计：
1. 核心配置（宽基指数）
2. 卫星配置（行业/主题）
3. 再平衡规则
4. AI辅助方式
```

### EP37-39 - 回测与系统化

```markdown
帮我检验投资系统的完整性：

请按以下清单检查我的系统：
□ 入场规则是否明确
□ 出场规则是否明确
□ 仓位规则是否明确
□ 风险预算是否明确
□ 复盘机制是否建立
□ 情绪管理是否有方案
□ 系统是否有版本管理

对每项缺失的，帮我补充。
```
