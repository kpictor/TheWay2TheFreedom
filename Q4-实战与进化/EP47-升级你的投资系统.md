## 💡 系统升级原则

### 何时升级

| 触发条件 | 升级内容 |
|----------|----------|
| 连续违反某条规则 | 规则可能不切实际，需要修改 |
| 发现新的风险点 | 增加防护措施 |
| 某策略明显失效 | 调整或移除 |
| 积累了足够数据 | 优化参数 |

### 升级注意事项

```
1. 不要因为一两次失败就改变系统
2. 变更要有充分理由，记录变更原因
3. 每次只改一个地方，便于追踪效果
4. 保留历史版本，便于回顾
```

### 版本控制模板

```markdown
## 投资系统变更日志

### v2.0 (日期)
**变更内容**：
- 修改了止损规则从10%改为8%

**变更原因**：
- 数据显示10%止损经常被触发后反弹
- 但8%能更好地控制损失

**预期效果**：
- 减少20%的止损次数

**实际效果**：（3个月后填写）
-
```
