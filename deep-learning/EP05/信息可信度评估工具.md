# EP05 - 信息可信度评估工具

> 实战分析工具：评估市场信息的可靠性

---

## 🎯 用前须知

**核心理念**：
> "不要平等对待所有信息。根据来源的可信度加权。" — 达利欧

**本工具用途**：
- 评估消息来源的可信度
- 分析信息背后的动机
- 判断信息是否已被市场消化
- 决定是否基于该信息行动

---

## 📋 工具1：信息来源评估表

### 每条重要消息需评估

| 评估维度 | 评分(1-5) | 说明 |
|----------|-----------|------|
| **来源可信度** | | 官方公告=5，匿名爆料=1 |
| **历史准确率** | | 该来源过去预测的准确性 |
| **利益相关性** | | 来源是否有利益冲突？越低越好 |
| **信息时效性** | | 第一手=5，转载数次=1 |
| **可验证性** | | 能否交叉验证？ |
| **总分** | /25 | |

**解读**：
- 20-25分：高可信度，可作为参考依据
- 15-19分：中等可信度，需交叉验证
- 10-14分：低可信度，仅作为背景了解
- <10分：忽略此信息

## 📋 工具2：信息动机分析

### 追问"为什么告诉我"

```
信息：________________________________

【来源分析】

1. 谁在传播这个信息？
   _______________________________________________

2. 他们的利益立场是什么？
   □ 利益相关方（公司/大股东）
   □ 中间人（分析师/媒体）
   □ 独立第三方
   □ 匿名来源

3. 如果这个信息被相信，谁会受益？
   _______________________________________________

4. 如果这个信息是假的，谁会受益？
   _______________________________________________

【真实意图推测】

5. 发布这条信息的可能目的：
   □ 合规披露（被迫）
   □ 引导预期（操控）
   □ 制造流量（变现）
   □ 分享见解（善意）
   □ 其他：________________

【结论】

6. 这条信息的可信度：□高 □中 □低
7. 是否值得据此行动：□是 □否 □需要更多验证
```

## 📋 工具3：信息时效性检查

### 判断信息是否"过期"

```
信息/事件：________________________________

【时间线重建】

1. 事件实际发生时间：_______________
2. 消息公开发布时间：_______________
3. 你得知的时间：_______________

【价格反应检查】

4. 事件发生日的价格变化：
   日期：_______ 涨跌幅：_______%

5. 消息公布日的价格变化：
   日期：_______ 涨跌幅：_______%

6. 你得知时的价格位置：
   相比事件前：□更高 □更低 □差不多

【结论】

□ 信息尚未反映：有行动空间
□ 信息部分反映：需谨慎
□ 信息已完全反映：不宜追入
□ 信息已过度反映：可能有反向机会
```

## 📋 工具4：逆向解读检查清单

### 看到利好/利空消息时

**看到"利好"消息时问**：
- [ ] 这个消息发布前，股价涨过了吗？
- [ ] 谁在发布消息后能获益？
- [ ] 这是"靴子落地"还是"超预期"？
- [ ] 知情者有没有已经买入？
- [ ] 这会吸引谁来接盘？

**看到"利空"消息时问**：
- [ ] 这个消息发布前，股价跌过了吗？
- [ ] 这是"已知的担忧"还是"新的风险"？
- [ ] 最坏的情况有多坏？
- [ ] 谁可能被迫卖出？
- [ ] 价格是否已经反映了这个利空？

## 💡 使用建议

1. **不需要分析每条信息**
   - 只分析你打算据此行动的重要信息
   - 小道消息不值得花时间

2. **建立个人信息源档案**
   - 长期追踪各来源的准确性
   - 定期更新评估

3. **先怀疑再相信**
   - 假设每条信息都有问题
   - 通过工具验证后再采信

4. **承认局限**
   - 即使分析正确，市场也未必按预期走
   - 工具帮助提高概率，不保证结果
