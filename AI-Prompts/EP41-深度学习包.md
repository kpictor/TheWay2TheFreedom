## 📋 第一步：设计你的交易日志

### 日志模板V1

```markdown
# 交易记录 #___

## 基本信息
- 日期：
- 标的：
- 方向：买入/卖出
- 价格：
- 数量：
- 仓位金额：
- 占总资产比例：%

## 入场/出场分析
- 入场/出场理由：
- 符合哪条规则：
- 不符合的地方：

## 情绪记录
- 交易前情绪（1-10）：
- 交易时感受：
- 交易后情绪（1-10）：

## 决策质量评估（交易结束后填）
- 结果：盈/亏 ___
- 做对了什么：
- 做错了什么：
- 运气成分：高/中/低
- 下次改进：

## AI助手点评（可选）
[粘贴AI的分析]
```

### Prompt A - 定制你的日志

```markdown
我想设计适合自己的交易日志模板。

我的情况：
- 交易频率：每周___次
- 交易风格：___
- 最想改进的方面：___
- 可用于记录的时间：每天___分钟

请帮我调整模板，使其：
1. 能在我的时间内完成
2. 覆盖我最需要关注的维度
3. 简洁但不遗漏关键信息
```

---

## 💪 第二步：坚持机制设计

### 如何确保记录不中断

```markdown
帮我设计交易日志的坚持机制：

**触发器**
- 什么时候记录？（交易后立即？收盘后？）
- 用什么工具记录？（App？Excel？纸质？）

**奖励**
- 连续记录7天的奖励？
- 月度复盘后的奖励？

**惩罚**
- 漏记一次的处理？
- 连续漏记的熔断？

**提醒**
- 设置日历提醒
- 和其他习惯绑定
```

---

## 🔬 第三步：日志分析

### Prompt B - 周度日志分析

```markdown
这是我本周的交易日志：
[粘贴日志内容]

请帮我分析：
1. 规则执行率是多少？
2. 有没有情绪化交易的证据？
3. 盈利交易和亏损交易的模式差异？
4. 需要改进的地方？
5. 做得好的地方？
```

### Prompt C - 月度模式识别

```markdown
这是我过去一个月的交易汇总：
[粘贴汇总数据]

请帮我识别：
1. 我的交易有什么模式？
2. 什么情况下表现好？
3. 什么情况下表现差？
4. 情绪和业绩的相关性？
5. 需要调整规则吗？
```
